Contexte du projet

Une startup FinTech souhaite développer un prototype très simple de portefeuille numérique (Digital Wallet) afin de comprendre le fonctionnement d’un serveur backend avant d’utiliser un framework comme Express.
Contenu fonctionnel attendu

Gestion des utilisateurs (CRUD)

Un utilisateur contient :

    id
    name 

Gestion des portefeuilles (CRUD)

Une portefeuille contient :

    id
    user_id
    name
    sold 

Chaque portefeuille doit être attribué à un utilisateur, avec un solde initial de 0 DH.

Opérations financières

Fonctionnalités :

    Déposer un montant sur un portefeuille
    Retirer un montant
    Garder le calcul du solde

Bonus

Faire l’historique de chaque opération et sauvegarder les données dans un fichier .json.

Règles métier obligatoires

    Le montant doit être strictement positif
    Le retrait est refusé si le solde est insuffisant
    L’utilisateur doit exister
    Les réponses doivent être en JSON
